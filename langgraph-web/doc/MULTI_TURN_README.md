# 作文批改多轮对话页面使用说明

## 📁 新增/修改的文件

### 1. **新增：EssayMultiTurnChat.vue**
- 支持多轮对话的 Vue 组件
- 功能：
  - 首次请求填写学生姓名和作文题目
  - 自动管理会话（sessionId）
  - 支持追问和连续对话
  - 显示会话状态
  - 可清除会话重新开始

### 2. **更新：essayReview.ts（类型定义）**
新增类型：
- `EssayChatRequest` - 多轮对话请求
- `EssaySession` - 会话信息

### 3. **更新：essayReview.ts（API）**
新增 API：
- `chatWithEssayReview()` - 多轮对话接口
- `getEssaySession()` - 获取会话信息
- `clearEssaySession()` - 清除会话

### 4. **更新：App.vue**
- 添加"作文批改（多轮对话）"标签页
- 默认显示多轮对话页面

## 🎯 页面功能

### 首次请求流程

1. **填写信息**
   - 学生姓名：例如"张三"
   - 作文题目：例如"生活中的美好"
   - 消息内容：例如"请批改这篇作文"

2. **发送请求**
   - 系统自动创建会话
   - 执行完整工作流（布置作业 → 提交作文 → 批改）
   - 流式输出评语

3. **会话创建成功**
   - 显示"会话中"徽章
   - 提示"会话已创建，您可以继续追问"
   - 表单隐藏，只显示消息输入框

### 追问流程

1. **继续对话**
   - 直接输入追问内容
   - 例如："能详细分析一下主题立意吗？"
   - 系统使用对话历史回答

2. **连续追问**
   - 可以无限次追问
   - 每次都会保留完整的对话历史
   - 例如：
     - "这篇文章有哪些优点和不足？"
     - "如何改进结尾部分？"
     - "请给出修改建议"

3. **清除会话**
   - 点击"清除会话"按钮
   - 清空所有对话记录
   - 重新开始新的批改流程

## 🔧 技术实现

### 1. 会话管理

```typescript
// 会话 ID
const sessionId = ref<string | null>(null)

// 是否有活跃会话
const hasSession = ref(false)

// 首次请求后自动保存 sessionId
if (!hasSession.value) {
  hasSession.value = true
}
```

### 2. 请求构建

```typescript
// 首次请求
const request: EssayChatRequest = {
  studentName: '张三',
  essayTopic: '生活中的美好',
  message: '请批改这篇作文'
}

// 追问
const request: EssayChatRequest = {
  sessionId: 'abc-123-def',  // 使用上次的 sessionId
  message: '能详细点吗？'
}
```

### 3. 流式输出

```typescript
await chatWithEssayReview(request, {
  signal: abortController.signal,
  onEvent: (data) => {
    // 实时更新消息内容
    assistant.content = data
    scrollToBottom()
  },
})
```

### 4. 清除会话

```typescript
function clearCurrentSession(): void {
  // 调用后端 API 清除
  clearEssaySession(sessionId.value)
  
  // 重置前端状态
  sessionId.value = null
  hasSession.value = false
  messages.value = []
  // ...
}
```

## 📊 数据流

### 完整对话数据流

```
用户填写表单
    ↓
构建 EssayChatRequest（包含 studentName、essayTopic、message）
    ↓
发送 POST /api/essay/chat
    ↓
后端创建会话，返回 sessionId
    ↓
前端保存 sessionId
    ↓
流式接收评语
    ↓
用户输入追问
    ↓
构建 EssayChatRequest（包含 sessionId、message）
    ↓
发送 POST /api/essay/chat
    ↓
后端使用对话历史回答
    ↓
流式接收回答
    ↓
（可以继续追问...）
    ↓
用户点击"清除会话"
    ↓
发送 DELETE /api/essay/session/{sessionId}
    ↓
重置所有状态
```

## ⚠️ 注意事项

### 1. 会话持久化
当前会话存储在内存中（ConcurrentHashMap），服务重启后会丢失。

**生产环境建议**：
- 使用 Redis 存储会话
- 设置过期时间（如 24 小时）
- 支持分布式部署

### 2. 会话数量限制
当前没有对话轮数限制。

**生产环境建议**：
- 限制最多 100 轮对话
- 超出后提示用户创建新会话

### 3. 错误处理
已实现基本的错误处理：
- 网络错误显示错误信息
- 支持停止生成
- 表单验证

### 4. 浏览器兼容性
支持现代浏览器：
- Chrome 80+
- Firefox 75+
- Safari 13+
- Edge 80+
